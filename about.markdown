<div class="lang-en">
  <p>PureScholar is built for researchers who want a homepage that reads like a curated research surface rather than a generic portfolio.</p>

  <h3>What changed in this refactor</h3>
  <ul>
    <li>The homepage is now separated cleanly from the layout shell.</li>
    <li>Profile, news, projects, and publications all come from YAML data files.</li>
    <li>The visual system now supports multiple switchable homepage styles without duplicating content templates.</li>
  </ul>

  <h3>Where to customize</h3>
  <ul>
    <li>Edit <code>_data/profile.yml</code> for identity, highlights, metrics, and research focus.</li>
    <li>Edit <code>_data/navigation.yml</code> to control homepage navigation and topic filters.</li>
    <li>Edit <code>_data/styles.yml</code> to rename, reorder, or extend the style picker.</li>
    <li>Edit <code>_data/news.yml</code>, <code>_data/projects.yml</code>, and <code>_data/publications.yml</code> for the main academic content.</li>
  </ul>
</div>

<div class="lang-zh">
  <p>PureScholar 面向希望把主页做成“研究信息面板”而不是通用作品集的学者。</p>

  <h3>这次重构完成了什么</h3>
  <ul>
    <li>首页内容已经和布局壳彻底分离。</li>
    <li>个人信息、动态、项目和论文统一由 YAML 数据文件驱动。</li>
    <li>视觉系统现在支持多种可切换风格，而且不需要重复维护内容模板。</li>
  </ul>

  <h3>后续主要改哪里</h3>
  <ul>
    <li>修改 <code>_data/profile.yml</code>，替换个人身份、重点信息、指标和研究方向。</li>
    <li>修改 <code>_data/navigation.yml</code>，调整首页导航和筛选项。</li>
    <li>修改 <code>_data/styles.yml</code>，重命名、排序或继续扩展风格选择器。</li>
    <li>修改 <code>_data/news.yml</code>、<code>_data/projects.yml</code>、<code>_data/publications.yml</code>，更新主页主体内容。</li>
  </ul>
</div>
